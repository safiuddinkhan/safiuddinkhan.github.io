What is the Purpose of Union Data Type in C Language

2022-05-13 21:25:01 +0500 PKT Technology Programming

OK, So recently I am getting back into development after quite a lot of time so I thought that I should practice my C language a bit. While practising my C language I came to the topic of structures and unions both of which at first glance seem quite similar and union seems to look as if it is out the place and useless but it is not and both are different.

struct safi{
int a;
int b;
int c;
};

union safi1{
int a;
int b;
int c;
};

struct safi s1;
s1.a = 5;
s1.b = 7;
s1.c = 6;

printf("struct -> s1.a = %d s1.b = %d s1.c = %d\n",s1.a,s1.b,s1.c);

union safi1 s2;
s2.a = 5;
s2.b = 7;
s2.c = 6;

printf("union -> s2.a = %d s2.b = %d s2.c = %d\n",s2.a,s2.b,s2.c);

At first glance both look similar but the first one will output:

struct -> s1.a = 5 s1.b = 7 s1.c = 6

And the second will output:

union -> s2.a = 6 s2.b = 6 s2.c = 6

Why it is as such?

The answer is simple while structure holds all values of its elements a union can only hold a single value at a time. Also, structure memory size is the sum of all its elements meanwhile in the case of a union its memory size is that of its largest element. In our case above the size of the union is 4 bytes as in equal to an int variable while that of our structure is 12 bytes as in equal to the sum of three int variables.

So why does the union data type exist in the C language?

If you look at it you can do a very low-level polymorphism with a union.

I hope you will finally understand as I have what is the purpose of a union and why it exists.
