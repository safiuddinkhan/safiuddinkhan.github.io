Virtualization in old x86 CPUs

2022-04-20 08:12:11 +0500 PKT Technology Windows Linux

We all know that modern x86 CPUs can do hardware virtualization but do you know a form of hardware-accelerated virtualization has existed in all x86 CPUs since the 1980s. It is called Virtual 8086 mode and it was first introduced in the 80386 CPUs to run applications made in real mode to run in the protected mode. This feature allowed Windows 3.x and above and similarly OS/2 to run multiple DOS applications to run both windowed and full screen. Similarly this feature was utilized by Windows NT line of operating systems by NTVDM though within a limited way and direct hardware access was not allowed. It was also used by DOSEMU in Linux to virtualize DOS applications.

Virtual 8086 mode does not exist in 64-bit long mode of modern x86_64 CPUs and that is the reason you cannot run DOS applications in all 64-bit version of Windows and DOSEMU has been depricated in all 64-bit versions of Linux but this feature can still be used in the 32-bit legacy mode of the modern x86_64 CPUs or within the Intel VT-x and AMD-V virtualization modes of modern x86_64 CPUs.
