How to run SuperTuxKart properly on your Raspberry Pi OS Bullseye

2022-03-23 12:08:06 +0500 PKT Technology Linux Raspberry Pi Gaming

OK, so you install SuperTuxKart on your Pi4/400 from the repos and find out that it does not run properly and you see missing graphics like in the screenshot and you say to yourself what a piece of crap that it cannot even run SuperTuxKart properly. Well, this is because most SBCs including Pi4 have the guts of mobile phones and they mainly support OpenGL ES properly and the implementation of OpenGL 3 is not complete.

One way to fix this and make SuperTuxKart playable is to force the game to use the legacy fallback drivers. To do that follow the following steps.

  1. Open the terminal and use your faviorite editor whether its nano or vim and open the file "/usr/share/games/supertuxkart/data/graphical_restrictions.xml" via sudo and locate the following line at the bottom "<card vendor="Broadcom" os="linux" disable="HighDefinitionTextures256"/>"

  2. Change ""HighDefinitionTextures256"" with ""ForceLegacyDevice"" like this "<card vendor="Broadcom" os="linux" disable="ForceLegacyDevice"/>" and save the file.

And now when you run the game the next time it will display the graphics properly and would be playable.
